# Main App (musait.app) Update Guide for OTP Migration

**Goal:** Update the main application to use the new Railway Worker for OTP generation instead of handling it internally.

## 1. Environment Variables

Add the following environment variables to your Vercel (or deployment) configuration for `musait.app`:

```env
# The public URL of your deployed Railway Worker
RAILWAY_WORKER_URL="https://worker-production.up.railway.app"

# The shared secret key for internal API communication
INTERNAL_API_KEY="your_internal_api_key_for_musait_app"
```

> **Note:** Ensure `INTERNAL_API_KEY` matches the value in your Railway Worker variables.

## 2. Replace OTP Generation Logic

Find the code responsible for generating OTPs. It is likely in an API route (e.g., `app/api/auth/otp/route.ts` or similar) or server action.

**Old Logic (to be removed):**
- Generating 6-digit code locally.
- Saving to `phone_verifications` table directly.
- Calling WhatsApp API directly.

**New Logic (Worker Proxy):**
- Call `POST /otp/request` on the Worker.

### Example Implementation

```typescript
// app/api/auth/otp/route.ts (or wherever your OTP request handler is)

const WORKER_URL = process.env.RAILWAY_WORKER_URL;
const API_KEY = process.env.INTERNAL_API_KEY;

export async function POST(req: Request) {
  const { phone, context } = await req.json();

  if (!phone) {
    return Response.json({ error: "Phone required" }, { status: 400 });
  }

  try {
    // Proxy request to Worker
    const response = await fetch(`${WORKER_URL}/otp/request`, {
      method: "POST",
      headers: {
        "Content-Type": "application/json",
        "x-api-key": API_KEY!, // Authenticate with worker
      },
      body: JSON.stringify({
        phone,
        context: context || "login",
      }),
    });

    if (!response.ok) {
      const errorData = await response.json();
      return Response.json(errorData, { status: response.status });
    }

    const data = await response.json();
    return Response.json(data); // { success: true, requestId: ... }

  } catch (error) {
    console.error("Worker OTP Request Failed:", error);
    return Response.json({ error: "Internal Server Error" }, { status: 500 });
  }
}
```

## 3. Callback Handling

Ensure your `/auth/callback` route is ready to handle magic links generated by the worker.

When a user clicks the magic link in WhatsApp, they will be redirected to:
`https://musait.app/auth/callback?token=...&type=magiclink`

The existing Supabase Auth callback handler should already support this if it uses `supabase.auth.verifyOtp({ token, type: 'magiclink', ... })`.

**Verify:**
- Check `app/auth/callback/route.ts`.
- Ensure it handles `type=magiclink`.

## 4. Database Cleanup (Optional)

Once migration is confirmed working:
- You can stop using the `phone_verifications` table in the main app.
- The worker uses `phone_login_codes` table.

## 5. Testing

1. Deploy the changes to `musait.app`.
2. Go to the login page and enter a phone number.
3. Check WhatsApp for the OTP code.
4. Reply with the code on WhatsApp.
5. Click the magic link in the response.
6. Verification successful!
